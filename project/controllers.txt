# 📁 Dossier `controllers/`

## 🎯 Rôle global

Le dossier `controllers/` sert de **pont entre les interfaces et le moteur**.

Son rôle est de :

* recevoir les actions de l’interface
* les transmettre proprement au moteur
* exposer une API simple aux interfaces

👉 Le controller ne contient pas la logique du jeu.
Il sert à **éviter que l’interface parle directement au moteur profond**.

---

## 📄 `game_controller.py`

### Rôle

`GameController` est la façade utilisée par :

* le CLI
* la GUI

Il expose des méthodes simples :

* démarrer une partie
* démarrer un tour
* annuler un tour
* résoudre une défense
* récupérer l’état actuel

### Position dans l’architecture

* au-dessus de `GameEngine`
* en dessous des interfaces

### Pourquoi c’est utile

Sans controller, les interfaces devraient :

* connaître directement `GameEngine`
* gérer plus de détails internes
* risquer de se coupler trop au moteur

Avec le controller :

* le CLI et la GUI restent plus simples
* le moteur garde son indépendance
* le projet est mieux structuré

### Évolution probable

Plus tard, le controller pourrait :

* exposer des méthodes plus adaptées aux interfaces
* gérer plus de conversions de données
* intégrer des commandes de persistance
* servir de point d’entrée pour undo / snapshots

### Améliorations possibles

* pas grand-chose à faire maintenant
* il est volontairement mince, et c’est une qualité

---

## 🧠 Conclusion du dossier `controllers/`

Le dossier `controllers/` est très simple, mais très sain.

👉 Il fait exactement ce qu’il doit faire :

* pas trop de logique
* pas de dépendance UI dans le moteur
* séparation claire entre interface et engine

---

## 🔎 À retenir

* `game_controller.py` = relais propre entre UI et moteur

---

## ⚠️ Point d’attention futur

Si la GUI devient plus riche ou si la persistance est branchée directement à l’interface, le controller pourrait gagner quelques responsabilités supplémentaires.

Mais pour la V3, il est à la bonne taille.

**Confiance : 0.99**

**Caveat :**

* un controller très fin peut parfois sembler “inutile”, mais ici il joue bien son rôle d’isolation architecturale.
