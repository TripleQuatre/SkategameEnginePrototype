Documentation V8 - Dossier `validation/`

Role global
`validation/` protege la coherence des donnees, des etats et des actions.

Fichiers importants

`config_validator.py`
- verifie la coherence de `MatchConfig`
- structure supportee
- families runtime supportees
- policies compatibles
- coherence preset/config

`action_validator.py`
- verifie si une action peut etre executee
- start turn
- resolve attack
- resolve defense
- cancel turn
- transitions de roster entre deux tours
- applique aussi les blocages de `Uniqueness` et `Repetition`

`state_validator.py`
- verifie la coherence interne de `GameState`
- phases et sous-phases
- indices d'attaquant/de defense
- coherence trick engage / trick absent
- bornes des compteurs de tentative

Invariants V8 importants
- `MatchConfig` pilote le runtime, pas un vieux ruleset central
- si aucun trick n'est engage :
  - la turn phase doit rester ouverte
  - les indices de defense actifs doivent etre remis a zero
- si un trick est engage en phase attaque :
  - le compteur d'attaque doit etre coherent
- si un trick est engage en phase defense :
  - les defenseurs doivent etre construits
  - le defenseur courant doit etre borne

Validation metier V8 importante
- un trick deja valide peut etre bloque par `Uniqueness`
- un trick trop souvent relance puis rate peut etre bloque par `Repetition`
- ces blocages utilisent l'identite canonique du dictionnaire quand elle existe

Point a retenir
`validation/` n'est pas un simple garde-fou technique.
Il encode aussi des decisions metier explicites du runtime actuel.
