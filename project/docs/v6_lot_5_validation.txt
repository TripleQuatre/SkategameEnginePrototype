SkateGame Engine Prototype - Validation Lot 5 V6

Role du document
Ce document resume ce qui a ete effectivement livre pour le lot 5 de la V6.

Statut
Lot 5 implemente.

Date de validation
2026-04-15

Rappel du role du lot 5
Le lot 5 sert a faire apparaitre la variante active dans l'historique,
sans encore demander une refonte d'affichage des interfaces.

Livraisons effectives

1. Contexte de match dans l'historique
`History` expose maintenant `build_match_context()`.

Cette methode permet de reconstruire un contexte lisible du match
a partir de l'evenement `GAME_STARTED`.

2. Nouveau type dedie
Le type `HistoryMatchContext` a ete ajoute dans `core/history.py`.

Il porte actuellement :
- `mode_name`
- `preset_name`
- `player_names`
- `turn_order`
- `starting_attacker_name`
- `initial_turn_order_policy`
- `attacker_rotation_policy`
- `defender_order_policy`

3. Enrichissement de `GAME_STARTED`
`GameFlow.start_game()` ecrit maintenant dans l'historique V6 :
- le `mode_name`
- le `preset_name`
- les policies actives

4. Branchement moteur
`GameEngine` transmet maintenant `match_parameters` a `GameFlow`
pour que l'evenement `GAME_STARTED` puisse embarquer ce contexte.

5. Compatibilite avec l'historique existant
La reconstruction des tours via `build_turns()` et `build_rows()`
n'a pas ete modifiee dans sa logique de base.
Le lot 5 ajoute du contexte,
il ne remplace pas l'historique des tours.

6. Preparation de test
`tests/unit/test_history.py` a ete enrichi avec un test cible
sur `build_match_context()`.

Ce que le lot 5 ne fait pas encore

1. Affichage interface du contexte
CLI et GUI n'affichent pas encore explicitement ce contexte V6.
Ce travail appartient au lot 6.

2. Historique de reconfiguration de match
Le lot 5 prepare l'historique des variantes initiales,
mais pas encore l'historique d'un futur `join between turns`.

3. Resume utilisateur formate
Le lot 5 fournit l'information structuree,
pas encore une presentation UX complete.

Resultat structurel
L'historique sait maintenant raconter non seulement les tours,
mais aussi le cadre V6 dans lequel la partie a ete lancee.

Conclusion
Le lot 5 a donne a la V6 un niveau minimal mais propre de tracage historique
de la variante active.
Le chantier peut maintenant passer au lot 6,
qui est l'exposition des presets V6 dans les interfaces.
