Documentation V8 - Dossier `interfaces/`

Role global
`interfaces/` contient les couches de presentation du moteur.

Principe
- aucune logique metier critique ne doit vivre ici
- les interfaces lisent le state et appellent le controller
- la logique de trick valide vient du dictionnaire, pas de l'interface

CLI

`interfaces/cli/cli_app.py`
- interface texte jouable
- permet :
  - nouvelle partie preset ou custom
  - chargement d'une sauvegarde
  - saisie assistee des tricks
  - choix obligatoire d'une suggestion
  - resolution d'attaque et de defense
  - annulation de tour
  - undo
  - save/load
  - affichage de l'historique

Choix V8 du CLI
- l'utilisateur peut taper librement un debut de trick
- le trick n'est valide qu'apres selection d'une suggestion
- l'interface affiche tricks complets et continuations valides
- le label affiche prefere la forme canonique issue du dictionnaire

GUI

`interfaces/gui/gui_app.py`
- interface Tkinter
- permet :
  - setup de partie
  - chargement d'une sauvegarde
  - saisie assistee des tricks avec barre de recherche et dropdown integre
  - blocage visuel si aucun trick terminal n'est selectionne
  - resolution d'attaque et de defense
  - undo
  - save/load
  - affichage de l'historique
  - affichage de `setup_details`

Choix V9.1 du GUI
- la selection de suggestion est explicite
- les boutons critiques restent bloques tant que la saisie n'est pas valide
- l'affichage prefere le label canonique
- la fenetre utilise un conteneur scrollable global
- le score `1v1` et `battle` passent par un rendu tabulaire compact
- la navigation clavier couvre les actions principales
- une vue `setup_details` expose la configuration active de la partie

Oracles GUI
- les attentes visibles du GUI sont formalisees dans `docs/code/v9_1_gui_oracles.txt`
- le simulateur GUI de V9.2 les utilise comme base d'attentes visibles

Etat V9.2
- CLI et GUI sont alignes sur le dictionnaire
- la saisie libre pure n'est plus le chemin normal
- le GUI est plus lisible et plus testable
- le GUI expose des cibles stables pour le harness
- le harness pilote et observe le GUI sans devenir une dependance metier
