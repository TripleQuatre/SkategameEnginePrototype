Documentation V8 - Regles et familles runtime

Important
La source de reference metier des regles reste dans `docs/sheets/`.
Ce document explique comment les regles sont structurees dans le code actuel.

Familles runtime reelles

1. Scoring
- vit dans `match/scoring/`
- actuellement :
  - `letters_scoring.py`
  - `scoring_factory.py`

2. Victory
- vit dans `match/victory/`
- actuellement :
  - `last_player_standing.py`
  - `victory_factory.py`

3. Fine rules
- vivent surtout dans :
  - `config/fine_rules_config.py`
  - `match/flow/trick_rules.py`
- actuellement :
  - `Uniqueness`
  - `Repetition`

4. Dictionary-side validity
- vit dans `dictionary/`
- ne decide pas si un trick est autorise dans le match
- decide si un trick existe et quelle est son identite canonique

Separation importante
- `dictionary/` dit : "ce trick existe-t-il ?"
- `match/flow/trick_rules.py` dit : "ce trick est-il autorise maintenant ?"

Etat V8
- `Uniqueness` compare les tricks valides canoniques
- `Repetition` compare les tricks lances puis rates
- les futures regles fines devront s'ajouter dans cette logique de famille,
  pas en recodant le moteur a la main
