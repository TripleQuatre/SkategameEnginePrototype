V9.3 Regression Library

Objective
This document records the dedicated regression scenarios introduced during V9.3.

These scenarios are intentionally narrower than the stress library:
- each one protects a concrete bug or a very specific fragile sequence
- each one is named after the failure it prevents from coming back
- they complement the broader stress scenarios instead of replacing them

Regression scenarios

1. deep_undo_restores_trick_input_regression
- protects the bug where undoing back to setup and restarting the game could
  leave trick entry in a broken state

2. load_restores_engaged_turn_controls_regression
- protects the flow where loading a saved engaged turn must restore the right
  match controls and keep invalid controls disabled

3. consultation_undo_restores_engaged_turn_regression
- protects end-of-game consultation mode and the undo path that exits it

4. player_count_rebuilds_fields_regression
- protects the custom setup flow where changing the player count must rebuild
  the corresponding player input fields

Design note
The regression library should stay small and concrete.
Future bugs found manually should be added here when they correspond to one
specific broken sequence, instead of being hidden only inside a larger stress
scenario.

V10/V10.1 additions

5. undo_restores_multiple_attack_change_controls_regression
- protects the case where undoing a multiple-attack trick change must restore
  the previous trick and keep the attacker-side change controls usable

6. switch_disabled_filters_suggestions_regression
- protects the filtering rule that switch tricks should disappear from
  suggestions when switch is disabled, instead of being suggested and rejected
  later

7. verified_switch_load_restores_confirmation_regression
- protects the load path where a pending verified switch must restore the
  dedicated normal verification controls

8. attack_repetition_synergy_blocks_invalid_setup_regression
- protects the setup validation that blocks incompatible Attack/Repetition pairs
  before the game starts

9. relevance_order_persistence_details_regression
- protects the persistence of `Order = relevance`, including the visible
  criterion and base order shown in setup details after load
