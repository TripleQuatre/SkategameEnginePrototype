SkateGame Engine Prototype - Informations V8

Statut
La V8 est une version structurante du moteur.

Cap V8
- introduire la feature `Dictionary`
- brancher la saisie de tricks sur une identite canonique
- livrer le premier dictionnaire `inline_primary_grind`
- ajouter `Uniqueness`
- ajouter `Repetition`
- conserver la base consolidée de V7.x

Objectifs atteints
- dictionnaire autonome dans `dictionary/`
- premiere implementation concrete `inline_primary_grind`
- saisie assistee CLI et GUI
- stockage canonique des tricks dans le runtime
- persistence des tricks canoniques
- `Uniqueness` sur tricks valides
- `Repetition` sur tricks lances puis rates
- docs V8 posees

Capacites utilisateur actuelles
- creer une partie preset ou custom
- jouer via CLI
- jouer via GUI
- saisir un trick via suggestions
- utiliser `Uniqueness`
- utiliser `Repetition`
- sauvegarder / charger / undo

Limites assumees
- sport reellement exploitable : `inline`
- dictionnaire reellement exploitable : `inline_primary_grind`
- pas encore de `Banned Tricks`
- pas encore de `Randomizer`
- pas encore de vraie regle `Switch` pour les combos
- interfaces encore utilitaires

Base de robustesse V8
- suite unitaire
- suite integration
- tests interfaces
- tests dictionnaire
- tests save/load
- tests undo
- tests invariants

Bilan
La V8 fait passer le projet d'un moteur configurable a un moteur capable de
raisonner sur des tricks canoniques, ce qui debloque les futures regles plus
avancees.
