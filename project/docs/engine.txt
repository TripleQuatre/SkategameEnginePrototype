Documentation V4 - Dossier `engine/`

Role global
`engine/` contient l'orchestration active du match.

Fichiers

`game_engine.py`
- facade principale du moteur
- initialise validation, flow, mode, snapshots et persistence
- expose en V4 :
  - `start_game`
  - `start_turn`
  - `resolve_defense`
  - `cancel_turn`
  - `undo`
  - `save_game`
  - `load_game`

`game_flow.py`
- coeur de l'orchestration
- gere les transitions de partie
- produit les evenements

Decisions V4 importantes dans `game_flow.py`
- `start_turn(trick)` engage le trick mais ne le consomme pas
- `cancel_turn(trick)` est possible seulement si aucun trick n'est engage
- le trick est consomme en fin terminale de tour
- la fin de partie nettoie correctement l'etat du tour courant

`turn_resolver.py`
- gere une tentative de defense
- succes
- echec avec essais restants
- echec final avec penalite

`end_conditions.py`
- determine les eliminations
- determine la fin de partie
- retourne le gagnant si applicable

Undo V4
- snapshots pris avant chaque action mutante du moteur
- restauration complete d'etat
- pas de rollback evenementiel manuel

Save / Load V4
- sauvegarde d'un etat complet
- rechargement valide et revalidation du mode
- remise a zero de l'historique d'undo apres chargement

Etat V4
Le dossier `engine/` est maintenant le vrai coeur stabilise du projet.
