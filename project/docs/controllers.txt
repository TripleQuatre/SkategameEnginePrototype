Documentation V8 - Dossier `controllers/`

Role
`controllers/` fournit la couche de relais entre les interfaces et la session
applicative.

Fichier principal

`game_controller.py`
- encapsule `GameSession`
- expose une API stable aux interfaces et aux tests

Methodes exposees
- `start_game()`
- `start_turn(trick)`
- `resolve_attack(success)`
- `resolve_defense(success)`
- `cancel_turn(trick)`
- `undo()`
- `save_game(filepath)`
- `load_game(filepath)`
- `get_state()`
- suggestions et resolution de tricks V8
- operations de transition de roster

Pourquoi cette couche reste utile
- le CLI et le GUI n'accedent pas directement au runtime detaille
- l'API utilisateur reste simple
- la session applicative garde son independance

Etat V8
- le controller reste volontairement fin
- il ne duplique pas la logique metier
- il expose les nouvelles capacites V8 sans les recoder

Point a retenir
Le controller est le point de contact unique des interfaces.
Il doit rester stable meme quand l'interieur du runtime continue d'evoluer.
