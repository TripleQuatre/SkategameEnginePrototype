Documentation V5 - Dossier `controllers/`

Role
Le dossier `controllers/` fournit la couche de relais entre les interfaces et le moteur.

Fichier principal

`game_controller.py`
- encapsule `GameEngine`
- expose une API simple aux interfaces

Methodes exposees en V5
- `start_game()`
- `start_turn(trick)`
- `resolve_defense(success)`
- `cancel_turn(trick)`
- `undo()`
- `save_game(filepath)`
- `load_game(filepath)`
- `get_state()`

Pourquoi cette couche reste utile
- le CLI et le GUI n'accedent pas directement a l'orchestration interne
- l'API utilisateur reste simple
- le moteur garde son independance

Etat V5
- le controller reste volontairement fin
- il n'ajoute pas de logique metier
- il suit l'evolution du moteur sans la dupliquer

Capacites exposees en V5
- duel et battle
- undo en fin de partie
- save/load de parties terminees
- consultation d'une partie rechargee

Point a retenir
Le controller est volontairement stable :
il sert de point de contact unique pour toutes les interfaces et pour les futurs outils
de simulation de scenarios.
