SkateGame Engine Prototype - Validation Lot 4 V6

Role du document
Ce document resume ce qui a ete effectivement livre pour le lot 4 de la V6.

Statut
Lot 4 implemente.

Date de validation
2026-04-15

Rappel du role du lot 4
Le lot 4 sert a faire vivre les nouveaux objets V6
dans la persistence du projet.

Livraisons effectives

1. Serialization des policies V6
`Serializer` sait maintenant :
- serialiser `MatchPolicies`
- deserialiser `MatchPolicies`

Les champs V6 persistants sont :
- `initial_turn_order`
- `attacker_rotation`
- `defender_order`

2. Evolution de `MatchParameters` dans la persistence
`serialize_match_parameters()` et `deserialize_match_parameters()`
gerent maintenant :
- `policies`
- `preset_name`

3. Retrocompatibilite de lecture
Le chargement des anciennes configurations reste supporte.
Si les champs V6 sont absents :
- `policies` retombe sur ses valeurs par defaut
- `preset_name` retombe a `None`

4. Preparation des tests de persistence V6
`tests/unit/test_persistence.py` a ete enrichi pour verifier :
- roundtrip de `preset_name`
- roundtrip de `policies`
- roundtrip de `rule_set` avec preset officiel
- presence des champs V6 dans le JSON
- lecture d'une ancienne config sans champs V6

Resultat structurel
La V6 ne se limite plus a une configuration en memoire :
ses nouveaux champs peuvent maintenant etre sauvegardes
et relus proprement.

Ce que le lot 4 ne fait pas encore

1. Affichage interface des presets
CLI et GUI ne presentent pas encore les presets V6.
Ce travail appartient au lot 6.

2. Tracage historique minimal des variantes
La persistence connait maintenant les variantes V6,
mais l'historique ne les raconte pas encore explicitement.
Ce travail appartient au lot 5.

3. Verification complete des tests de persistence en environnement local
Les tests ont ete prepares pour la V6,
mais ils n'ont pas ete reroules ici dans la foulee.

Raison
Le plan utilisateur etait de continuer l'implementation sans pause test large,
et l'environnement avait deja montre un probleme d'acces
sur certains tests `pytest` utilisant `tmp_path`.

Ce que le lot 4 debloque
Le chantier peut maintenant entrer dans le lot 5 :
faire apparaitre la variante active dans l'historique.

Conclusion
Le lot 4 a rendu la persistence compatible avec la base de configuration V6,
tout en gardant la lecture des anciennes donnees possible.
