SkateGame Engine Prototype - Informations V5

Statut
La V5 est une version d'extension structurelle du moteur.

Cap V5
- introduire le multijoueur via `battle`
- garder le moteur independant des interfaces
- fiabiliser la fin de partie
- rendre save/load compatible avec plusieurs modes
- commencer une vraie stabilisation des interfaces par tests

Objectifs atteints
- mode `battle` pour parties a `3+` joueurs
- `turn_order` persistant dans l'etat
- orchestration des tours deleguee aux modes
- `TURN_FAILED` remplace `TURN_CANCELLED`
- undo possible en fin de partie dans la session courante
- sauvegarde et chargement d'une partie terminee
- chargement adaptatif du bon mode via `GameSave`
- historique reconstruit en tours + sous-lignes defenseur
- CLI et GUI compatibles `one_vs_one` et `battle`
- premiers tests d'interfaces
- premier runner de scenarios reutilisable

Capacites utilisateur actuelles
- creer une partie `one_vs_one`
- creer une partie `battle`
- jouer une partie via CLI
- jouer une partie via GUI
- undo pendant la partie et apres la fin de partie
- sauvegarder une partie en cours ou terminee
- charger une partie en cours ou terminee
- consulter l'historique d'une partie finie

Limites assumees
- presentation GUI encore utilitaire
- details visuels non prioritaires
- tests GUI encore cibles et non exhaustifs
- pas encore de systeme complet pour couvrir automatiquement toutes les combinaisons de regles futures
- pas de gestion des equipes

Ce que la V5 ne cherche pas encore a faire
- architecture reseau
- couche sociale / app mere
- presets communautaires complets
- systeme de points alternatif
- modes equipes

Base de robustesse V5
- unit tests
- integration tests
- tests CLI
- tests GUI cibles
- save/load sequences
- undo sequences
- state invariants
- simulation invariants
- scenario runner

Bilan
La V5 fait passer le projet d'un moteur duel stabilise a un moteur d'arbitrage multi-mode
deja utilisable, persistable et testable.
