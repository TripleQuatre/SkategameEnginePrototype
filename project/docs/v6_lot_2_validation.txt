SkateGame Engine Prototype - Validation Lot 2 V6

Role du document
Ce document resume ce qui a ete effectivement livre pour le lot 2 de la V6.

Statut
Lot 2 implemente.

Date de validation
2026-04-15

Rappel du role du lot 2
Le lot 2 sert a valider la coherence des combinaisons V6 supportees
des le niveau configuration.

Livraisons effectives

1. Validation des modes connus
`ConfigValidator` rejette maintenant un `mode_name` inconnu.

2. Validation mode / nombre de joueurs
Les regles deja presentes sont conservees :
- `one_vs_one` exige exactement 2 joueurs
- `battle` exige au moins 3 joueurs

3. Validation des policies pour `one_vs_one`
Le validateur rejette maintenant pour `one_vs_one` :
- une politique d'ordre initial randomisee
- un ordre des defenseurs inverse

Cette restriction reste coherente avec le modele reel actuellement supporte.

4. Validation des presets officiels
Si `preset_name` est renseigne :
- le preset doit exister dans `PresetRegistry`
- `mode_name` doit correspondre au preset
- `policies` doivent correspondre au preset
- `rule_set` doit correspondre au preset

5. Rejet des incoherences explicites
Le validateur attrape maintenant plus tot :
- preset inconnu
- preset connu mais configuration effective differente
- mode inconnu

Tests ajoutes ou renforces
`tests/unit/test_validation.py` couvre maintenant explicitement :
- mode inconnu
- incoherence `one_vs_one` / policies
- preset officiel valide
- preset inconnu
- preset incoherent avec `policies`
- preset incoherent avec `rule_set`

Verification effectuee
- `tests/unit/test_validation.py`
- `tests/unit/test_config_models.py`

Resultat
Ces tests passent.

Ce que le lot 2 ne fait pas encore

1. Support moteur des policies
Les modes et le flow moteur ne consomment pas encore les nouvelles policies V6.
Ce travail appartient au lot 3.

2. Persistence complete des nouveaux champs
La serialisation des nouveaux champs V6 n'est pas encore le sujet de ce lot.
Ce travail appartient au lot 4.

3. Validation fine de toutes les futures combinaisons
Le lot 2 pose un garde-fou initial.
Il ne pretend pas encore formaliser toute la combinatoire future du moteur.

Decision implicite importante
Dans l'etat actuel, si `preset_name` est fourni,
il signifie "ce match correspond exactement a ce preset officiel".

Si plus tard on veut permettre une configuration derivee d'un preset officiel,
il faudra probablement :
- soit retirer `preset_name`
- soit introduire un mecanisme distinct de "preset de base"

Ce que le lot 2 debloque
Le chantier peut maintenant entrer dans le lot 3 :
integration des policies V6 dans le moteur et les modes.

Conclusion
Le lot 2 a donne au projet un premier vrai garde-fou de configuration V6.
Les incoherences principales sont maintenant attrapees avant l'execution du moteur.
