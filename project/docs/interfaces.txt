Documentation V5 - Dossier `interfaces/`

Role global
`interfaces/` contient les couches de presentation du moteur.

Principe
- aucune logique metier critique ne doit vivre ici
- les interfaces lisent le state et appellent le controller
- elles servent surtout a tester et valider le moteur

CLI

`interfaces/cli/cli_app.py`
- interface texte jouable
- permet en V5 :
  - nouvelle partie `one_vs_one` ou `battle`
  - chargement d'une sauvegarde au lancement
  - saisie et confirmation de tricks
  - resolution de plusieurs defenseurs
  - annulation de tour via `TURN_FAILED`
  - undo
  - save/load pendant la partie
  - affichage de l'historique
  - mode de consultation en fin de partie

Choix V5 du CLI
- nombre de joueurs dynamique
- mode infere automatiquement :
  - `2` joueurs -> `one_vs_one`
  - `3+` joueurs -> `battle`
- historique branche sur `build_turns()`
- mode de cloture avec :
  - undo
  - save
  - history
  - load
  - new game
  - quit

GUI

`interfaces/gui/gui_app.py`
- interface Tkinter minimale
- permet en V5 :
  - setup dynamique du nombre de joueurs
  - nouvelle partie `one_vs_one` ou `battle`
  - chargement d'une sauvegarde depuis l'ecran d'accueil
  - saisie et annulation de trick
  - resolution de defense
  - undo
  - save/load pendant la partie
  - affichage de l'historique
  - consultation d'une partie finie
  - retour rapide a `New game`

Choix V5 du GUI
- affichage score compatible multijoueur
- description du tour compatible defenseurs multiples
- historique branche sur `build_turns()`
- mode consultation explicite en `Phase.END`

Etat V5
- les interfaces sont alignees avec `battle`
- elles exposent correctement la fin de partie
- elles restent volontairement utilitaires

Sujet a garder pour la suite
- tests d'interface plus riches
- snapshots texte CLI
- scenarios GUI plus nombreux
- corrections visuelles ponctuelles si necessaire
