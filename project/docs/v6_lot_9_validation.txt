SkateGame Engine Prototype - Validation Lot 9 V6

Role du document
Ce document resume ce qui a ete effectivement livre pour le lot 9 de la V6.

Statut
Lot 9 implemente.

Date de validation
2026-04-15

Rappel du role du lot 9
Le lot 9 ajoute le retrait de joueur entre deux tours,
en miroir du lot 8, avec comme cas officiel
`battle -> one_vs_one`.

Livraisons effectives

1. Nouvelle operation moteur
`engine/game_engine.py` expose maintenant
`remove_player_between_turns(...)`.

Cette operation :
- refuse le retrait hors phase `TURN`
- refuse le retrait si un trick est deja engage
- refuse le retrait si moins de deux joueurs resteraient
- retire proprement le joueur du roster
- reconstruit `turn_order`
- reattribue immediatement l'attaquant si le joueur retire etait l'attaquant
- bascule `battle -> one_vs_one` quand il ne reste plus que deux joueurs

2. Reconfiguration de configuration effective
`MatchParameters` est reconfigure correctement pendant l'operation :
- mise a jour de `player_ids`
- mise a jour de `mode_name` quand un battle redevient duel
- invalidation du `preset_name` des que la composition du match change
- reinitialisation des `policies` par defaut du duel quand on revient a `one_vs_one`

3. Undo compatible reconfiguration
Comme pour le lot 8, `undo()` restaure correctement :
- le roster precedent
- le mode precedent
- le preset precedent
- l'attaquant precedent

4. Tracage de l'operation
Un nouvel evenement `PLAYER_REMOVED` est ajoute a l'historique.

Il permet de tracer :
- le joueur retire
- le mode avant/apres
- le preset avant/apres
- le roster courant
- le `turn_order` courant
- l'attaquant courant apres reattribution

5. Historique contextuel mis a jour
`build_match_context()` sait maintenant tenir compte de `PLAYER_REMOVED`
pour refleter la configuration active apres retrait.

6. Surface publique
`controllers/game_controller.py` expose aussi
`remove_player_between_turns(...)`.

7. Interfaces utilisables
La fonctionnalite est maintenant accessible depuis :
- le CLI via la commande `/remove`
- le GUI via le bouton `Remove player`

Dans les deux cas, le retrait reste refuse si un trick est deja engage.

Couverture ajoutee

Des tests ont ete ajoutes pour couvrir :
- retrait de joueur entre deux tours
- bascule `battle -> one_vs_one`
- reattribution de l'attaquant apres retrait
- rejet du retrait pendant un trick engage
- undo apres retrait
- persistence du match reconfigure
- restauration du contexte historique
- integration CLI et GUI

Verification locale

Les verifications executees dans cet environnement ont confirme :
- les parcours moteur / controller / undo / historique
- les parcours CLI et GUI

Les tests `save/load` utilisant `tmp_path` restent bloques ici
par une permission Windows sur le dossier temporaire `pytest`,
pas par une regression fonctionnelle du moteur.

Conclusion
La V6 sait maintenant reconfigurer le roster dans les deux sens
entre deux tours : ajout et retrait. Le cas officiel
`battle -> one_vs_one` fonctionne, y compris quand le joueur retire
etait l'attaquant courant.
