Name
Order

Type
rule

Ensemble
Fundamental Rules

Parent Rule
None

Definition
Fundamental rule that defines the base attack order of players for the whole match.

Purpose
Determine in which order players take the attacker role according to a stable initial logic chosen before the match starts.

Options
Relevance: the attack order is built from a fixed relevance criterion applied to the selected player profiles.
Examples of V10 criteria:
- alphabetical
- age
- experience_time
- local_rank

Random: the attack order is chosen randomly at the start of the match.

Choice: the attack order is chosen manually during setup.

Activation Conditions
None

Dependencies
player list
available player-profile criteria when Relevance is used

Interactions
Attack Order
Defense Order
Advantage
Clash
Round

Default Behavior
One attack order is defined at match start and kept as the base order for the rest of the match.

Advanced Behavior
This base order may later be reused or modified by advanced order rules.
Relevance is not a separate rule: it is one option of Order.

Code Impact
turn order
player rotation
match initialization
round logic

UX Notes
Must be configured early in the setup flow.
If Relevance is selected, the interface must clearly expose the chosen criterion.
If Choice is selected, the interface must allow users to reorder players explicitly.

Open Questions
Need to confirm which fixed relevance criteria are included in the first implementation.
