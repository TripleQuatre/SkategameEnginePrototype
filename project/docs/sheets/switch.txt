Name
Switch

Type
rule

Ensemble
Special Rules

Parent Rule
None

Definition
Specific rule that defines in which conditions switch tricks are allowed during the match.

Purpose
Control whether switch tricks are forbidden, freely allowed, or subject to prior validation conditions.

Options
Enabled (option): all switch tricks are allowed.
Normal (option): to impose a switch trick, the same trick must already have been validated in its normal form during the match by any player.
Verified (option): any player may attempt a switch trick at any time, but when the attacker validates the switch they must also prove the normal version through a separate normal verification confirmation. This proof does not count as a launch attempt and is not stored as its own validated trick in history.
Disabled (option): no switch trick is allowed.

Activation Conditions
The active sport or discipline must treat switch as a meaningful variation.

Dependencies
Attack

Interactions
Uniqueness
Attack
Tolerance
Out

Default Behavior
Switch authorization depends entirely on the selected option.

Advanced Behavior
Normal adds a match-level prerequisite on the normal version of the trick.
Verified adds a second confirmation linked to attack validation:
- Normal verified: the switch validation stands.
- Normal not verified: the switch attempt is invalidated.
The normal proof does not consume an additional attack attempt by itself; only the switch attempt is counted.
This rule should only be exposed in sports where switch is relevant.

Code Impact

UX Notes
Must only appear when the active sport makes switch relevant.
The difference between Normal and Verified must be explained simply in the interface.
If Switch is disabled, switch tricks should not appear in suggestions.
If Switch is normal, only switch tricks whose normal form has already been validated in the match should appear in suggestions.

Open Questions
Later, the product may refine how the verified normal proof is rendered in history beyond the initial attack-column marker.
