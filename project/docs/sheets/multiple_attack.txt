Name
Multiple Attack

Type
rule

Ensemble
Advanced Attack & Defense

Parent Rule
Attack

Definition
règle avancée qui définit si l’attaquant peut changer de trick au cours de ses différentes tentatives pendant un même tour

Purpose
préciser si plusieurs tentatives d’attaque permettent de rejouer le même trick ou d’en tenter un autre

Options
Disabled (option) : par défaut, le lanceur ne peut pas changer de figure au cours des essais suivant le premier
No Repetition (option) : par défault, les essais effectués pendant un même tour d’attaque n’affectent pas la règle Repetition comme des répétitions distinctes
Enabled (option) : le lanceur peut changer de figure au cours de ses différentes tentatives d’attaque

Activation Conditions
Attack doit être supérieur à 1

Dependencies
Attack
Repetition

Interactions
Attack
Repetition
Uniqueness

Default Behavior
le lanceur ne peut pas changer de trick pendant son tour d’attaque

Advanced Behavior
l’option No Repetition modifie la manière dont les essais d’un même tour sont comptabilisés pour Repetition

Code Impact
résolution des tentatives d’attaque
historique des tricks tentés
contrôle de répétition

UX Notes
ne doit apparaître que si Attack est supérieur à 1

Open Questions
il faudra préciser plus tard comment traiter les variantes proches d’un même trick dans un même tour
Pour No Repetition, il faudrait suggerer de caller le nombre d’attack avec le nombre de répétition de tel sorte qu’un attaquant n’arrive jamais à court de Repetition avant d’avoir utiliser toutes ses tentatives d’attaque
