Name
Multiple Attack

Type
rule

Ensemble
Advanced Attack & Defense

Parent Rule
Attack

Definition
Advanced attack rule that defines how an attacker may reuse or change tricks across multiple attack attempts during the same attack turn.

Purpose
Clarify what happens when Attack is greater than 1:
- whether the attacker must keep the same trick across all attack attempts
- whether the attacker may switch to another trick from the second attack attempt onward
- how Repetition is counted across several attack attempts made during the same attack turn

Options
Disabled (main mode, default):
the attacker must keep the same trick during the whole attack turn.
Repetition is counted normally, launch by launch.

No Repetition (linked modifier, on/off):
this modifier changes only how Repetition is counted during one attack turn.
When enabled, all failed attempts made on the same trick during the same attack turn count as one single repetition.
This modifier may be enabled even if Multiple Attack is Disabled.

Enabled (main mode):
from the second attack attempt onward, the attacker may choose another trick to impose.
Repetition is still counted normally unless No Repetition is also enabled.

Activation Conditions
Attack must be greater than 1.
If Attack is 1 or lower, this rule must not appear in the UI and has no gameplay effect.

Dependencies
Attack
Repetition

Interactions
Attack
Repetition
Uniqueness

Default Behavior
Multiple Attack = Disabled
No Repetition = Off

Behavior Details
Disabled + No Repetition Off:
the attacker keeps the same trick during the whole attack turn.
Each failed launch counts normally for Repetition.

Disabled + No Repetition On:
the attacker keeps the same trick during the whole attack turn.
Several failed launches on that trick during the same attack turn count as one repetition.

Enabled + No Repetition Off:
from the second attack attempt onward, the attacker may switch to another trick.
Each failed launch counts normally for Repetition.

Enabled + No Repetition On:
from the second attack attempt onward, the attacker may switch to another trick.
For each trick, several failed launches during the same attack turn count as one repetition.

Example
Repetition = 4
Multiple Attack = Disabled
No Repetition = Off
If the attacker tries and fails Makio 4 times, Makio consumes 4 repetitions.

Repetition = 4
Multiple Attack = Disabled
No Repetition = On
If the attacker tries and fails Makio 4 times during the same attack turn, Makio consumes 1 repetition.

UX Notes
This rule must appear as one single UI block only when Attack > 1.
The UI should present:
- a main mode with Disabled / Enabled
- a linked No Repetition toggle

Product Notes
Conceptually, Multiple Attack and No Repetition are not exactly the same behavior.
However, they must stay linked in product/UI because they both belong to the same advanced attack block.

Open Questions
Need to define precisely how No Repetition interacts with edge cases involving trick variants judged identical or different by Repetition.
