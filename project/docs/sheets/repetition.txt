Name
Repetition

Type
rule

Ensemble
Attack & Defense

Parent Rule
None

Definition
Rule that defines how many times a trick already failed by the attacker may still be attempted during the match.

Purpose
Control how failed tricks may be retried by the attacker across the match.

Options
Choice (option): defines how many times the same trick may still be retried, with 3 as the default reference value.
Common (option): the same repetition limit is shared across players.
Disabled (option): removes the repetition cap on failed tricks.

Activation Conditions
None

Dependencies
history of failed tricks

Interactions
Uniqueness
Attack
Multiple Attack

Default Behavior
A failed trick may still be retried a limited number of times according to the chosen value.

Advanced Behavior
Common makes the repetition limit shared across players.
When Multiple Attack introduces No Repetition, several failed attempts on the same trick during the same attack turn may count as one single repetition.
When Attack is greater than 1, Multiple Attack is Disabled, and No Repetition is Off, Repetition may also be constrained to remain compatible with full attack turns.

Code Impact
trick history
action validation
repetition checks

UX Notes
Must remain clearly distinct from Uniqueness.
The difference between:
- a failed trick that may still be retried
- and a validated trick that may not be imposed again
must stay explicit.

Open Questions
Need to define later how Repetition interacts with complex trick variants judged identical or different.
