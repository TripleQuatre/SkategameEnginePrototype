Documentation V5 - Logique des donnees

But
Ce document decrit la logique de donnees la plus importante du moteur, independamment
des interfaces.

Source de verite
- `GameState` est la source de verite principale
- `History` est une trace exploitable du passe
- `Snapshot` est une image restaurable de l'etat
- `GameSave` est une enveloppe persistable de partie

Distinction cle V5

1. Etat courant
- ce que le moteur est en train de vivre
- porte par `GameState`

2. Historique
- ce qui s'est produit
- porte par `History`

3. Snapshot
- ce qui peut etre restaure rapidement
- porte par `Snapshot` et `SnapshotHistory`

4. Sauvegarde
- ce qui peut etre stocke et recharge
- porte par `GameSave`

Cycle de tour
- tour ouvert : `current_trick is None`
- tour engage : `current_trick is not None`
- tour termine : resultat terminal produit

Donnees sensibles du state
- `turn_order`
- `attacker_index`
- `defender_indices`
- `current_defender_position`
- `defense_attempts_left`
- `current_trick`
- `validated_tricks`
- `history`

Semantique des tricks
- un trick peut etre propose et accepte pour entrer dans le tour
- il devient alors `current_trick`
- il n'entre dans `validated_tricks` qu'en fin terminale du tour

Semantique du multijoueur
- `turn_order` porte l'ordre complet du match
- `defender_indices` porte les defenseurs du tour courant
- un defenseur deja resolu peut devenir inactif sans invalider l'etat
- les defenseurs restants doivent rester actifs

Pourquoi c'est important
Cette separation permet a la V5 de garder coherents :
- le battle
- l'annulation de tour
- l'undo
- le save/load
- les tests d'invariants

Persistence
- la serialisation conserve la difference entre :
  - un trick engage
  - un trick deja consomme
  - un ordre de rotation actif
  - une partie complete sauvegardee

Conclusion
La V5 a etendu le modele de donnees sans perdre la clarte apportee par la V4.
