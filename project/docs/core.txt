Documentation V8 - Dossier `core/`

Role global
`core/` contient les structures fondamentales du moteur. Il reste le socle
commun sur lequel se branchent `match/`, `validation/`, `persistence/` et
`application/`.

Fichiers importants

`player.py`
- representation d'un joueur
- identifiants
- etat actif/inactif
- score expose pour compatibilite

`player_score.py`
- representation extensible du score
- permet de distinguer les lettres et les points
- prepare les futures variantes de scoring

`state.py`
- etat runtime canonique de la partie
- contient notamment :
  - joueurs
  - phase et turn_phase
  - ordre de tour
  - attaquant courant
  - defenseurs courants
  - trick courant
  - historique
  - tricks valides
  - donnees canoniques V8 des tricks

Semantique actuelle de `GameState`
- `current_trick is None` : aucun trick engage
- `current_trick is not None` : un trick est engage dans le tour courant
- `validated_tricks` : vue texte legacy des tricks consommes
- `validated_trick_data` : vue canonique V8 des tricks valides
- `failed_attack_trick_data` : historique canonique des tricks lances puis rates

`events.py`
- structure des evenements runtime
- `Event.name` s'appuie sur `EventName`

`history.py`
- stocke l'historique brut des evenements
- reste la trace narrative de la partie

`snapshots.py`
- `Snapshot` capture un state complet, plus la config si besoin
- `SnapshotHistory` gere la pile de snapshots pour l'undo

`types.py`
- enums partages du runtime
- phases
- statuts de resolution
- noms d'evenements

Point a retenir
`core/` ne porte pas la logique de match detaillee, mais il porte tout ce que
le moteur doit pouvoir restaurer, serialiser, valider et relire proprement.
