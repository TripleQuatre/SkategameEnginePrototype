Documentation V5 - Dossier `core/`

Role global
`core/` contient les structures fondamentales du moteur.

Fichiers importants

`player.py`
- representation d'un joueur
- score, activite, identifiants

`state.py`
- etat canonique de la partie
- contient :
  - joueurs
  - phase
  - `turn_order`
  - attaquant courant
  - defenseurs courants
  - trick courant
  - historique
  - ruleset
  - tricks consommes

Semantique V5 de `GameState`
- `current_trick is None` : tour ouvert
- `current_trick is not None` : tour engage
- `validated_tricks` : tricks definitivement consommes par la partie
- `turn_order` : ordre de rotation de la partie, persistant

`events.py`
- structure des evenements moteur
- `Event.name` est un `EventName`

`history.py`
- stocke la liste brute des evenements
- reconstruit maintenant :
  - `build_turns()`
  - `build_rows()` pour compatibilite temporaire
- modele V5 :
  - `HistoryTurn`
  - `HistoryDefenseRow`

`snapshots.py`
- `Snapshot` capture un etat complet
- `SnapshotHistory` gere une pile de snapshots pour l'undo
- reste la base de l'undo interne

`types.py`
- enums partages
- contient notamment :
  - `Phase`
  - `DefenseResolutionStatus`
  - `EventName`

Vocabulaire V5 important
- `TURN_FAILED` remplace `TURN_CANCELLED`
- un tour rate reste un vrai evenement de partie

Point a retenir
La V5 a fait evoluer le coeur moteur pour supporter le multijoueur sans abandonner
la lisibilite de l'etat ni la reconstruction d'historique.
