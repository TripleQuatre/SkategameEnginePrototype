Documentation V8 - Dossier `config/`

Role global
`config/` regroupe les modeles de configuration du match et les outils qui
permettent de passer du setup utilisateur au runtime reel.

Centre actuel
- `MatchConfig` est la source de verite runtime
- `MatchSetup` est la forme pratique de creation d'une partie
- `MatchParameters` subsiste surtout pour compatibilite

Fichiers importants

`match_config.py`
- configuration runtime canonique
- compose :
  - structure
  - attack
  - defense
  - scoring
  - victory
  - fine_rules

`match_setup.py`
- forme simple utilisee au moment du setup
- contient les champs principaux necessaires a une nouvelle partie

`setup_translator.py`
- traduit `MatchSetup` vers `MatchConfig`
- reconstruit aussi des vues de compatibilite vers `MatchParameters`

`fine_rules_config.py`
- porte les regles fines V8
- actuellement :
  - `uniqueness_enabled`
  - `repetition_mode`
  - `repetition_limit`

`match_preset.py`
- definition d'un preset officiel
- assemble structure, ruleset et regles fines

`preset_registry.py`
- catalogue des presets officiels

Autres familles de config
- `attack_config.py`
- `defense_config.py`
- `scoring_config.py`
- `victory_config.py`
- `structure_config.py`
- `match_policies.py`

Validation
- `validation/config_validator.py` verifie :
  - nombre de joueurs
  - coherence de structure
  - compatibilite des policies
  - familles runtime supportees
  - coherence preset/config

Etat V8
- la configuration est beaucoup plus structuree qu'en V5
- le runtime n'est plus centre sur un simple `rule_set`
- les futures regles et familles doivent s'ajouter dans cette architecture
  familiale, pas dans une liste plate de drapeaux
