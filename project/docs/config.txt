Documentation V5 - Dossier `config/`

Role
Le dossier `config/` regroupe les structures de configuration necessaires au lancement
d'une partie.

Fichiers

`match_parameters.py`
- decrit les parametres d'une partie
- contient :
  - `player_ids`
  - `mode_name`
  - `rule_set`
- sert de point d'entree au `GameEngine`

`rule_set_config.py`
- decrit les regles configurables du jeu
- contient actuellement :
  - `letters_word`
  - `elimination_enabled`
  - `defense_attempts`

`game_config.py`
- reste un emplacement de configuration plus globale
- pas central dans la V5 actuelle

Validation
- `ConfigValidator` verifie les parametres avant lancement
- nombre minimum de joueurs
- mode present
- compatibilite entre `mode_name` et nombre de joueurs
- mot de lettres non vide et borne
- nombre d'essais coherent

Etat V5
- la couche config reste simple
- elle supporte maintenant `one_vs_one` et `battle`
- elle reste volontairement legere

Ou la config intervient
- creation du moteur
- creation du controller
- lancement d'une nouvelle partie via CLI ou GUI
- sauvegarde et rechargement via `GameSave`

Points pour la suite
- presets de configuration
- options de match plus riches
- parametres de mode plus detailes
- futures variantes de scoring / end conditions
